胶囊存证如何工作

当你封存一颗时光胶囊或预言胶囊时，系统会将胶囊内容生成不可逆的数字指纹，写入区块链：

📬
封存胶囊
你的文字、图片等内容 + 封存时间
→
🔐
SHA-256 指纹
内容生成唯一指纹，任何改动都会导致指纹完全不同
→
⛓
写入区块链
指纹提交至 FISCO BCOS 链，区块之间链式锁定

为什么很难篡改？ 区块链的每个区块都包含前一个区块的哈希。修改任何历史数据，指纹就对不上，后续区块全部断裂。你可以在胶囊详情页查看存证信息，自行计算 SHA-256 验证指纹是否吻合。

存证了什么

时光胶囊

胶囊的完整内容（文字、图文）+ 创建时间，序列化后计算 SHA-256 哈希上链。

预言胶囊

「预测的事情」+「我的结论」+ 创建时间，以语义化字段上链，便于解封后对照验证。

外部锚定：证明链本身的时间线

以上机制保障了链上数据的完整性。但这条链由拾芳自托管运维，节点尚未对外开放——链上数据理论上可被质疑。外部锚定通过将关键哈希提交到独立的公共系统，提供不可篡改的时间证明。

三层互证架构

1
FISCO BCOS 区块链 胶囊内容指纹存证，完整的交易历史和区块链式哈希
2
GitHub 公开仓库 每日自动提交区块哈希，形成公开可审计的时间线
3
OpenTimestamps → Bitcoin 区块哈希锚定至比特币网络，获得密码学级别的时间证明

每日锚定流程

获取 FISCO BCOS 最新区块哈希
↓
提交至 OpenTimestamps 日历服务器
↓
记录写入公开仓库并自动推送
↓
OTS 服务器在 Bitcoin 出块后完成锚定

一个区块哈希隐式承诺了该链上从创世块到当前块的所有交易——覆盖所有胶囊。

能证明什么，不能证明什么

诚实声明

本机制能证明：你的胶囊内容在封存时间点确实存在，且此后未被篡改。

本机制不能证明：内容是否如实反映了你当时的真实想法。当所有节点由同一运营方控制时，运营方理论上可以先构造数据再上链锚定，时间线仍然成立。

这一局限性只能通过引入不受运营方控制的独立节点来消除。

当前价值与演进路径

外部锚定锁定了时间线，为将来引入独立节点或公开链数据时，证明历史记录不是"事后补的"提供了基础。当独立第三方节点加入网络后，信任链条将完全闭合。

如何验证

独立验证步骤

1. 在胶囊详情页查看存证哈希（content_hash）和区块号（block_number）

2. 使用胶囊原始内容自行计算 SHA-256，比对哈希是否一致

3. 安装 opentimestamps-client，运行 ots verify 验证对应区块的 OTS 时间证明

详细步骤请参阅仓库 README。
